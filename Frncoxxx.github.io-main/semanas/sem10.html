    SEMANA 10: JSP (Jakarta Server Pages) y API REST con Spring Boot

    Esta semana se trabajó con Apache Tomcat como servidor web para aplicaciones Java, y se introdujo JSP (Jakarta Server Pages) para generar páginas dinámicas combinando HTML + código Java. También se revisó el flujo de procesamiento de JSP (traducción a servlet, compilación y ejecución) y se practicó con sintaxis JSP (scriptlets, declaraciones, expresiones, directivas y acciones), además del uso de objetos implícitos como request, response, out y session. Finalmente, se inició el desarrollo de una API REST con Spring Boot aplicando estructura MVC, conexión a MySQL y pruebas con Postman.

    Lo que vimos en clase
    Apache Tomcat
    • Tomcat funciona como contenedor de servlets y JSP.
    • Carpetas clave: bin, conf, logs, webapps, work/temp.
    • Archivo principal: server.xml (Server, Service, Connector, Engine, Host, Context).
    • Parámetros típicos: port, maxThreads, connectionTimeout, redirectPort.
    JSP (Jakarta Server Pages)
    • Lenguaje script del lado servidor para construir páginas web dinámicas.
    • Se procesa con un motor JSP: intercepta solicitudes y genera la respuesta.
    • Flujo: JSP → Servlet → Compilación → Ejecución → HTML final.
    • Acceso a APIs Java (incluye JDBC para bases de datos).
    Sintaxis JSP: elementos principales
    Scriptlet

    <% código Java %>

    Permite incluir Java directo: variables, condicionales y bucles.

    Declaración

    <%! variables o métodos %>

    Define métodos o variables que luego se usan en la página.

    Expresión

    <%= expresión %>

    Imprime el resultado en el HTML renderizado.

    Directivas, Acciones y Objetos implícitos

    Se revisaron directivas como <%@ page %> (configuración de la página), <%@ include %> (inclusión en compilación) y <%@ taglib %> (bibliotecas de etiquetas). También se vieron acciones JSP con sintaxis XML <jsp:action_name ... /> para controlar el comportamiento del motor. Además, se practicó con objetos implícitos como request, response, out, session, application, config, pageContext y exception.

    Objeto ¿Para qué sirve? Ejemplo de uso
    request Leer datos de la solicitud (GET/POST) request.getParameter("numero")
    out Imprimir salida en la página out.println("Hola")
    session Guardar datos por sesión (visitas, usuario, etc.) session.setAttribute("contador", 1)
    Ejercicios de laboratorio (JSP)

    En el laboratorio se desarrollaron páginas JSP para:
    • Mostrar la fecha y hora actual en pantalla usando <%= new java.util.Date() %>.
    • Usar formularios con método GET y POST, recuperando datos con request.getParameter().
    • Implementar un ejemplo de cálculo de factorial validando datos del usuario y mostrando resultados con out.println().
    • Contar cuántas veces se ejecuta la página en una sesión usando session como contador de visitas.

    Introducción a Spring Framework y Spring Boot

    Se introdujo Spring Framework como un framework robusto para aplicaciones empresariales, destacando conceptos como Inversión de Control (IoC) y Inyección de Dependencias (DI). Spring usa un contenedor (Spring Container) que crea y administra objetos llamados beans, controlando su ciclo de vida. Luego, se revisó Spring Boot como herramienta para crear proyectos Spring de forma rápida, con configuración automática, servidor embebido, métricas y health checks.

    API REST con Spring Boot (MVC + MySQL + Postman)

    Se siguió una guía para crear una API con Spring Boot usando estructura MVC (controllers, models, repositories y services). Se configuró conexión a MySQL en application.properties y se construyó un ejemplo CRUD con la entidad Estudiante, usando Spring Data JPA para el repositorio y un controlador REST para crear, listar, actualizar y eliminar. Finalmente, se validó el CRUD mediante pruebas en Postman.

    Reflexión

    Aprendí que JSP permite generar páginas dinámicas en Java, pero su verdadero poder aparece cuando se combina con un servidor como Tomcat y conceptos de sesión y formularios. También entendí que Spring simplifica el desarrollo empresarial mediante IoC y DI, porque el framework se encarga de crear y administrar objetos. Finalmente, con Spring Boot pude ver una ruta clara para construir APIs REST con CRUD real (MySQL + JPA) y validarlas profesionalmente con Postman.
