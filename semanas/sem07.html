SEMANA 7: Hooks

1. Introducción a Hooks:

Los Hooks fueron introducidos en React 16.8 y permiten que los componentes de tipo función gestionen el estado y otras características sin necesidad de convertirlos en componentes de clase.

2. Tipos de Hooks:

- useState: Permite crear y actualizar el estado en un componente funcional.
- useEffect: Ejecuta efectos secundarios en componentes, como la obtención de datos o la manipulación del DOM.
- useContext: Permite compartir datos entre componentes sin necesidad de pasar props de un componente a otro.
- useReducer: Una alternativa más robusta a useState para manejar estados complejos.
- useRef, useNavigate, useParams: Otros hooks útiles para gestionar referencias, navegación y parámetros de URL.

Ejemplo: useState
Código import React, { useState } from 'react'; function Counter() { const [count, setCount] = useState(0); return ( <div> <p>Contador: {count}</p> <button onClick={()=> setCount(count + 1)}>Incrementar</button> </div> ); }
3. Gestión de Estado:

- useState y useReducer se utilizan para gestionar el estado local y complejo de una aplicación.
- useEffect maneja efectos secundarios como el manejo de APIs o temporizadores, y se ejecuta en diferentes fases del ciclo de vida de un componente (montaje, actualización).

4. Custom Hooks:

Se recomienda usar hooks personalizados para encapsular lógica reutilizable y mejorar la organización del código.
Los hooks personalizados deben seguir buenas prácticas como mantener el estado mínimo necesario y asegurar que las dependencias se gestionen correctamente en useEffect.
