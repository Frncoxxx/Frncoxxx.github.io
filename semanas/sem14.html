SEMANA 14: Desarrollo Backend con Python (Flask + MySQL + Plantillas)

En esta semana se trabajó el desarrollo backend con Python, revisando cómo un servidor como Apache puede ejecutar aplicaciones Python mediante mod_wsgi, y reforzando fundamentos del lenguaje: instalación, ejecución, sintaxis y buenas prácticas PEP-8. Luego se introdujo el framework Flask para construir aplicaciones web ligeras con rutas, plantillas y conexión a base de datos. También se explicó el motor de plantillas Jinja2 para generar HTML dinámico y separar lógica (backend) de presentación (frontend).

Lo que vimos en clase

• Apache HTTP Server: request/response y entrega de contenido web.
• Instalación portable de Apache (Apache Lounge) y configuración básica en httpd.conf.
• ¿Qué es mod_wsgi y para qué sirve: integrar Python (Flask/Django) dentro de Apache.
• Instalación de Python y verificación de versión (python --version).
• Gestión de paquetes con PIP (pip install, pip list, requirements.txt).
• Estándares de estilo con PEP-8 (indentación, nombres y organización de imports).
• Fundamentos de sintaxis Python: variables, condicionales, bucles, funciones y POO.
• Flask: rutas con @app.route, request/response y render_template.
• Jinja2: variables, condicionales, bucles, herencia de plantillas y filtros.

Flask y Jinja2

Flask es un microframework en Python para construir aplicaciones web de forma rápida y flexible. Permite definir rutas con @app.route, manejar formularios con request y devolver respuestas. Para la interfaz, Flask utiliza plantillas con Jinja2, que permite incrustar variables ({{ ... }}), condicionales y bucles en HTML, además de herencia de plantillas para reutilizar estructuras comunes.

Ejercicios de Laboratorio

Ejercicio 01 (Python - POO): Se implementó una aplicación orientada a objetos con una clase base Persona y una clase derivada Estudiante, aplicando herencia, uso de super() y sobrescritura de métodos (polimorfismo).

Ejercicio 02 (Flask + MySQL): Se creó una aplicación web en Flask que muestra un formulario HTML para registrar estudiantes y guarda los datos en una base de datos MySQL mediante la librería pymysql. Se definieron rutas: una para mostrar el formulario y otra para recibir el POST y ejecutar el INSERT en la tabla.

Reflexión

Esta semana entendí cómo Python puede usarse como backend real en aplicaciones web. Me quedó claro que Flask es ideal para proyectos ligeros porque permite definir rutas y conectar con bases de datos de forma sencilla. Además, el uso de Jinja2 ayuda a separar la lógica del servidor de la parte visual, haciendo el código más ordenado. Finalmente, con mod_wsgi aprendí que también se pueden desplegar aplicaciones Python dentro de Apache, integrándose con un entorno de servidor web tradicional.
